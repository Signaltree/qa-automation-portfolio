## What I do
Review documentation quality and coverage across the project:

### README
- Project purpose is clear
- Quick start guide (install, configure, run)
- Usage examples
- Tech stack listed
- Badges (CI status, coverage, license)

### Code Documentation
- Public API has JSDoc/TSDoc comments
- Complex logic has inline explanations
- Types/interfaces document their fields
- Deprecated items are marked

### Test Documentation
- Test suite structure is documented
- Test data sources are explained
- Running specific test categories is clear
- CI/CD pipeline is documented

### Operational Docs
- Deployment instructions
- Environment variable reference
- Troubleshooting guide
- Contribution guidelines (CONTRIBUTING.md)

## When to use me
Use this skill when creating a new project, before open-sourcing, or during documentation sprints.

## Documentation quality checklist
- [ ] README explains what and why
- [ ] Quick start works in < 5 steps
- [ ] API is documented (types, params, returns)
- [ ] Environment variables are documented
- [ ] CI/CD process is described
- [ ] Troubleshooting section exists
