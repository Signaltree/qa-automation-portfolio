## What I do
- Analyze the project's directory structure and layering
- Verify that the Page Object Model pattern is applied correctly
- Check separation of concerns between tests, pages, fixtures, and utils
- Validate that the config supports cross-browser, mobile, and CI workflows
- Suggest improvements to project organization

## When to use me
Use this skill when starting a new project, adding a new test framework, or restructuring the project. I help ensure the architecture is scalable, maintainable, and follows industry standards.

## Key questions I answer
- Is the project structure consistent with QA automation best practices?
- Are Page Objects properly separated from test logic?
- Does the configuration support all required environments?
- Is the project ready for CI/CD integration?
