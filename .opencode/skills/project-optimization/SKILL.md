## What I do
Analyze the project holistically and identify optimization opportunities:

### Build & Bundle
- Tree-shaking effectiveness
- Code splitting opportunities
- Bundle size analysis
- Build time optimization
- Dead code elimination

### Runtime Performance
- Critical rendering path
- Lazy loading opportunities
- Caching strategy (browser, CDN, server)
- Database query optimization
- Memory usage patterns

### Developer Experience
- Startup time (dev server, test runner)
- Hot reload effectiveness
- Test execution time
- CI pipeline duration
- Local development ergonomics

### Codebase Health
- Tech debt identification
- Deprecated API usage
- Migration roadmap for outdated patterns
- Consistency across modules
- Configuration duplication

## When to use me
Use this skill for quarterly project health checks, before scaling efforts, or when performance issues are reported.

## Optimization checklist
- [ ] Build time < 30s
- [ ] Test suite < 5min
- [ ] Bundle size within budget
- [ ] Lazy loading implemented for routes
- [ ] Caching headers configured
- [ ] No known tech debt hotspots
